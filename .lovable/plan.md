## 一、可行性评估

| 需求 | 可行性 | 关键风险与对策 |
|---|---|---|
| **1. 代码纠错（语法+逻辑）** | ✅ 已完成 MVP | DeepSeek 已接入，支持 6 种语言、统一/分栏 Diff、行内跳转 |
| **2. Python 财务案例库 + 模拟数据 + 模型嵌入** | ⚠️ 中等 | 浏览器内运行 Python 需 Pyodide（首加载 ~6MB）；复杂金融模型（如蒙特卡洛）建议在边缘函数侧由 AI 生成数据 + 前端图表展示，不必真跑 Python |
| **3. 课前-课中-课后闭环 + 分层推送** | ✅ 可行 | 需持久化（Lovable Cloud），需"教师/学生"双角色；分层算法用规则即可（正确率 + 用时 + 错误类型加权）|
| **4. 学生画像与报告** | ✅ 可行 | 雷达图用 Recharts；画像由 AI 基于操作记录汇总生成 PDF/可分享报告 |

**整体结论：技术全部可行**。建议把"Python 真执行"降级为"AI 模拟执行 + 真实图表"，可将开发周期从 8 周压缩到 4-5 周。

## 二、完善后的项目路线图（6 个里程碑）

### ✅ M1 — 设计骨架与代码诊断 MVP（已完成）
- 原研哉风格设计系统、CodeMirror 编辑器、6 语言切换、文件上传
- DeepSeek 结构化分析、评分卡、错误卡片
- 优化前后 Diff（统一/分栏）+ 行号一键跳转

### 🔜 M2 — 数据底座 & 角色（启用 Lovable Cloud）
- 启用 Cloud，建立表：`profiles`、`user_roles`(student/teacher)、`cases`、`case_categories`、`submissions`、`assignments`、`reports`
- 邮箱登录 + 角色守卫；学生 / 教师两套布局入口

### M3 — Python 财务案例库（15 个核心案例）
- 五大分类：财务报表分析｜量化投资｜风险管理｜税务计算｜财务预测
- 案例详情页：背景描述 → "一键生成"模拟数据（AI 生成 CSV/JSON）→ 在线编辑参考代码 → 运行结果
- "点点点"模式：选择数据规模/参数 → 自动拼装代码 → 直接出图

### M4 — 模型计算嵌入
- 内置模型：DCF 估值、Black-Scholes、VaR (历史/参数法)、CAPM、杜邦分析
- 模型作为"积木"嵌入案例：表单输入 → 边缘函数计算 → Recharts 可视化
- AI 解读结果（自然语言点评）

### M5 — 课前 / 课中 / 课后闭环
- **课前**：教师发布案例包，学生预习清单 + 知识点卡片
- **课中**：实时练习面板（计时、提示、AI 即时点评）
- **课后**：基于本次表现自动生成三级任务
  - 基础（巩固薄弱点）｜进阶（同类变式）｜挑战（综合应用）
  - 分层规则：`score < 60 → 基础`，`60-85 → 进阶`，`>85 → 挑战`

### M6 — 学生画像与报告
- 画像维度：语法掌握、逻辑思维、性能意识、金融建模、坚持度（5 维雷达图）
- 数据来源：所有 submissions 聚合 + AI 汇总
- 输出：在线报告页 + PDF 导出 + 教师班级看板（按维度排序、热力图）
